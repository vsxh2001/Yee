# Phase 1.3.1.1 — Numerical 2-D Cross-Section Eigensolver for Wave Ports

**Status:** Draft
**Owner:** TBD
**Phase:** 1.3.1.1
**Depends on:** Phase 1.3.1.0 (analytic TE10 rectangular waveguide wave-port, shipped at `47e6b51`), Phase 1.0 (mom-001, shipped)
**Blocks:** Tight microstrip wave-port S-parameter validation; mom-002 / mom-003 wave-port driven cases; CPW / GCPW / coplanar-pair port models.

## Assumption being challenged

`crates/yee-mom/src/ports.rs` ships two `ModalDistribution` variants today: `Uniform` (TEM-like placeholder, bit-for-bit equivalent to a delta-gap) and `Te10(RectangularWaveguideTe10)` (closed-form rectangular-waveguide TE10 from Pozar §3.3). Both are useful, neither is sufficient.

Production PCB designs that a Yee user actually wants to simulate — 50 Ω microstrip on FR-4, grounded CPW with via fences, asymmetric coplanar pairs, slot lines on alumina — are **not rectangular waveguides**. There is no closed-form Pozar formula for the dominant quasi-TEM mode on any of these. Hammerstad-Jensen synthesis equations exist for canonical microstrip, but they are curve-fits to numerical eigensolves, not eigensolves themselves; they break for thick conductors, non-uniform substrates, or anything inhomogeneous.

The fix is to compute the mode profile **numerically** on a 2-D triangular mesh of the cross-section. This is the standard 2.5-D method-of-moments workflow (driven 3-D problem hangs on a 2-D mode profile at the port boundary) and is what every commercial tool does. The mathematics — vector Helmholtz eigenproblem with Nedelec edge elements — is textbook (Jin 3rd ed. §8.5); the engineering risk is sparse-eigensolve plumbing, not physics.

## Scope

In:

- 2-D finite-element vector eigensolve on an externally supplied `TriMesh2D` of the port cross-section.
- Curl-conforming Nedelec edge elements for transverse `E_t`; nodal Lagrange elements for `E_z`. Mixed `(E_t, E_z)` formulation as in Lee/Sun/Cendes 1991.
- Dominant quasi-TEM (or quasi-TE) mode only — one eigenpair per `solve` call. Shift-and-invert Arnoldi targets the smallest-magnitude `β²` deficit, i.e. the mode closest to free-space propagation.
- Outputs: real `β` (propagation constant, rad/m), complex `Z_w` (wave impedance, Ω), and a cached complex mode profile `(E_t, E_z)` sampled per-mesh-element.
- Integration with `WavePort` via a new `NumericalCrossSection` modal distribution.
- Per-frequency call: the eigensolve is dispersive and must be re-run per swept frequency; that's the user's loop, the solver caches one frequency at a time.

Out:

- Higher-order modes (only the dominant mode is extracted; multi-mode S-matrix de-embedding is Phase 1.3.2).
- Full-wave hybrid modes with tensor permittivity / permeability (anisotropy is Phase 1.3.3+).
- Frequency-swept eigensolve in one call (caller drives the sweep).
- Dispersive material models inside the cross-section (Drude/Lorentz/Debye live in `yee-fdtd` and are not wired into the MoM port path).
- Mode-tracking across frequency (mode-crossing detection is a Phase 1.3.4 concern).
- 2-D meshing of the cross-section — the `TriMesh2D` is an input contract, generated by the caller (Gmsh, hand-rolled fixtures, or whatever the user has).

## Approach

The vector Helmholtz eigenproblem on a guided-wave cross-section, after separating `e^{-j β z}` propagation, reads

```
∇_t × (1 / μ_r) ∇_t × E_t  −  k_0² ε_r  E_t  =  −β² E_t
```

with the standard Nedelec mixed formulation introducing `E_z` to satisfy the divergence constraint inside dielectric regions. Discretize on a `TriMesh2D` of the port cross-section:

- `E_t` ↔ Whitney-1 (Nedelec edge) basis functions, one DoF per interior edge.
- `E_z` ↔ Whitney-0 (nodal Lagrange) basis functions, one DoF per interior vertex.
- PEC walls: Dirichlet on `E_t` tangential, Dirichlet on `E_z`.
- PMC / open boundaries: natural Neumann (omit the boundary integral term).

This produces a sparse generalized eigenproblem `A x = β² B x` with `A`, `B` real-symmetric (lossless case) or complex-symmetric (lossy). Solve via shift-and-invert Arnoldi targeting `σ ≈ β_guess²` where `β_guess = ω √(ε_r,max μ_r,max) / c` (the largest possible propagation constant in the cross-section).

Wave impedance for a quasi-TEM mode follows from the energy ratio:

```
Z_w  =  V / I  =  ( ∫_C E_t · dℓ )  /  ( ∮_∂S H_t · dℓ )
```

where `C` is a path from the signal conductor to the reference and `∂S` encloses the signal conductor. For arbitrary cross-sections this integration is also numerical and uses the Nedelec interpolation of the solved mode.

- Lee, Sun, Cendes, "Full-wave analysis of dielectric waveguides using tangential vector finite elements", IEEE T-MTT 1991, 39(8).

## Public API

Add a new variant to `ModalDistribution` and a builder to `WavePort`. The struct caches one frequency's solution.

```rust
/// Numerical 2-D cross-section mode, computed by FEM eigensolve.
///
/// Phase 1.3.1.1: dominant quasi-TEM / quasi-TE mode only. Call
/// [`Self::solve`] once per frequency before the wave-port RHS is built.
pub struct NumericalCrossSection {
    /// 2-D triangular mesh of the port cross-section.
    pub mesh: TriMesh2D,
    /// Per-triangle complex relative permittivity.
    pub eps_r: HashMap<TriId, Complex64>,
    /// Per-triangle complex relative permeability.
    pub mu_r: HashMap<TriId, Complex64>,
    /// Most recently solved frequency (NaN before first solve).
    solved_freq_hz: f64,
    /// Cached propagation constant β at `solved_freq_hz`.
    beta: f64,
    /// Cached wave impedance Z_w at `solved_freq_hz`.
    z_w: Complex64,
    /// Cached mode profile: per-edge tangential E, per-vertex longitudinal E.
    mode_profile: ModeProfile,
}

impl NumericalCrossSection {
    /// Build with empty caches; eigensolve is deferred to [`Self::solve`].
    pub fn new(
        mesh: TriMesh2D,
        eps_r: HashMap<TriId, Complex64>,
        mu_r: HashMap<TriId, Complex64>,
    ) -> Self;

    /// Run the 2-D eigensolve at the given frequency. Idempotent and
    /// caches; calling twice at the same frequency is a no-op.
    pub fn solve(&mut self, freq_hz: f64) -> Result<(), yee_core::Error>;

    pub fn beta(&self) -> f64;
    pub fn wave_impedance(&self) -> Complex64;
}

pub enum ModalDistribution {
    Uniform,
    Te10(RectangularWaveguideTe10),
    /// Phase 1.3.1.1: arbitrary cross-section via 2-D FEM eigensolve.
    Numerical2D(NumericalCrossSection),
}

impl WavePort {
    pub fn with_numerical_cross_section(mut self, mode: NumericalCrossSection) -> Self {
        self.modal_distribution = ModalDistribution::Numerical2D(mode);
        self
    }
}
```

`WavePort::rhs` gains a `ModalDistribution::Numerical2D` match arm that samples `mode.e_tangential_at(p)` at the port-edge midpoint (analogous to the existing `Te10` arm calling `mode.e_y_profile`).

## Definition of done

1. `NumericalCrossSection::new` and `::solve` exist; `cargo build -p yee-mom` clean.
2. `ModalDistribution::Numerical2D` variant exists; `WavePort::with_numerical_cross_section` builder exists.
3. **Validation case A — rectangular waveguide TE10.** Feed the numerical eigensolver a `TriMesh2D` of a WR-90 cross-section (22.86 × 10.16 mm, air-filled). At 10 GHz, the numerical solver's `β` must match `RectangularWaveguideTe10::beta(10e9)` within `0.1%` relative; `Z_w` must match `wave_impedance(10e9)` within `1%` magnitude; the L2 norm of the difference between the numerical `E_t` profile and the analytic `sin(π x / a)` must be `< 1%` of the analytic L2 norm.
4. **Validation case B — non-canonical cross-section.** WR-90 with a 1 mm-thick PEC septum inserted vertically at `x = a/2`. Internal sanity check: at 10 GHz, the numerical `β` must be **larger** than the analytic free-space `k_0` (slow-wave / loaded-guide effect; the septum forces the mode into a smaller effective cross-section). No closed-form reference, so the gate is the inequality `β_numerical > k_0` plus a regression-tracked numerical value.
5. Eigensolve runs in `< 10 s` wall-time per call on a 500-triangle mesh in `--release`. (Informational; not a CI gate.)
6. Verification chain green: `cargo build`, `cargo clippy -- -D warnings`, `cargo test --release`, `cargo fmt --check` all pass on `-p yee-mom`.
7. No regression on the mom-001 dipole gate (the eigensolver is opt-in; the default `WavePort` with `ModalDistribution::Uniform` is bit-for-bit unchanged).
8. `cargo doc --no-deps -p yee-mom` warning-free.

## Escape hatch

If `arpack-rs` (the natural sparse-eigensolve binding) is unmaintained, build-broken, or pulls in a system LAPACK that breaks CI's lint-clean policy, fall back to a **dense Hermitian eigensolve** via `nalgebra`'s `SymmetricEigen` on a coarse cross-section mesh (≤ 500 DoFs). Document the fallback in `crates/yee-mom/validation/README.md` and the `NumericalCrossSection` doc comment; the validation gates above (rectangular-waveguide TE10 within 0.1% / 1%) can still be met on a coarse mesh because the canonical case is well-conditioned.

If the dense fallback is also blocked (e.g. CI memory limits at 500 DoFs), surface and stop: the right next step is then a hand-written shift-and-invert via inverse-power-iteration on a `faer::sparse::SparseColMat`, which is more work than this sub-project promises to deliver and should be its own spec.
