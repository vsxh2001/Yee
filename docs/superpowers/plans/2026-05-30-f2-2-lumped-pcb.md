## Steps
1. `src/board.rs`: `Footprint{Smd0402,Smd0603,Smd0805}`+`pad()->PadSpec` (IPC land
   patterns: 0402 pad ~0.6×0.6mm pitch ~0.5mm; 0603 ~0.8×0.9 pitch ~1.6mm; 0805
   ~1.2×1.3 pitch ~2.0mm — use standard values, document source), `BranchKind`,
   `Placement`, `LumpedBoard`, `lumped_board(...)`. Signal line y = mid; ground
   rail y = 0; series footprints in-line (break + bridge), shunt footprints on a
   stub line→rail. Pads = `Polygon::rect` copper; ref-des L1,C1,…; record
   `Placement`s; assemble `Layout` (bbox from all polygons, ports at line ends).
   Document all public items.
2. `lib.rs`: re-export `Footprint, BranchKind, Placement, LumpedBoard, lumped_board`.
3. `tests/lumped_pcb_001.rs` per DoD 3 (placement count 2N, unique ref-des, pad
   count, **no pad overlap** (rect-disjoint test), finite positive bbox containing
   pads, branch y-coords).

## Escape hatch
Blocked > 30 min (placement geometry overlaps unavoidably at the chosen pitch; a
yee-layout accessor missing; series-vs-shunt routing to ground is ambiguous for
the skeleton) → STOP + surface the geometry + the overlap. Do NOT touch
F2.0/F2.1/dimension logic; do NOT add deps; do NOT pull yee-export in (export is
exercised elsewhere — this gate validates the Layout geometry).

## Done when

`git diff --stat <base>..HEAD` = only `crates/yee-filter/**`; F2.0/F2.1/dimension
untouched; WASM-safe.
