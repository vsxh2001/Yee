## Goal

Retire `fem_eig_006_magnitude_bounded` `|S_{11}(30 GHz)| < 0.1` by
replacing CFS-PML on the +x face with a TE_{10} wave-port
termination (W1 from spec §3). The W1 hypothesis is the simplest
viable fix: ~10-line driver change, no `yee-fem` API churn, no new
types, reusing the Phase 4.fem.eig.2 E2 wave-port machinery already
exercised by fem-eig-004 / fem-eig-005.

Four-step ladder T1-T4 lands in a single merge train:

1. **T1** — switch the fem-eig-006 driver's +x face from PML
   truncation (`FaceKind::Pec` after PML extension) to
   `FaceKind::WavePort(1)`. Remove `extend_mesh_with_pml` and
   `.with_cfs_pml(...)` calls; add a second `PortDefinition` for
   the +x port.
2. **T2** — smoke test compiles + runs without crashing.
   `cargo test -p yee-validation --release fem_eig_006_smoke_runs`
   exits 0; `fem_eig_006_no_nan_inf` exits 0.
3. **T3** — un-ignore `fem_eig_006_magnitude_bounded`; verify
   `|S_{11}(30 GHz)| < 0.1`. Run
   `cargo test -p yee-validation --release` end-to-end to confirm
   no regressions on fem-eig-003.
4. **T4** — tutorial note in
   `docs/src/tutorials/07-fem-open-cavity.md` on the v3.5.3
   wave-port termination; ROADMAP refresh.

CPU-only, single-threaded, scalar FP64 complex. No GPU. No new
dependencies. Same execution model as v3.5.2.

## Step T1 — driver switch from CFS-PML to wave-port termination

**Lane:** `crates/yee-validation/src/lib.rs`.

Two surgical changes inside
`run_fem_eig_006_high_aspect_pml_with_config`:

### T1.a — replace mesh extension + face classification

Currently (lines ~3792-3833):

```rust
let (mesh, pml_classes, _faces) =
    extend_mesh_with_pml(&cavity, &[PmlAxis::XMax], pml_config.thickness_cells)?;

// ... placeholder solver and centroids ...

let mut face_kinds: Vec<FaceKind> = Vec::with_capacity(centroids.len());
let tol = 1e-9;
for c in &centroids {
    let kind = if c.x.abs() < tol {
        FaceKind::WavePort(0)
    } else {
        FaceKind::Pec
    };
    face_kinds.push(kind);
}
```

Replace with:

```rust
// Phase 4.fem.eig.3.5.3 W1: cavity is its native (16, 3, 2) shape,
// no PML extension. The +x face becomes a second wave-port
// terminating the TE_{10} mode (Jin §10.6); the v3.5.2 CFS-PML
// path is retired here per ADR-0046.
let mesh = cavity;

// ... placeholder solver and centroids over the un-extended mesh ...

let mut face_kinds: Vec<FaceKind> = Vec::with_capacity(centroids.len());
let tol = 1e-9;
for c in &centroids {
    let kind = if c.x.abs() < tol {
        FaceKind::WavePort(0)
    } else if (c.x - FEM_EIG_006_A_M).abs() < tol {
        FaceKind::WavePort(1)
    } else {
        FaceKind::Pec
    };
    face_kinds.push(kind);
}
```

The `extend_mesh_with_pml` call goes away; `pml_classes` is no
longer needed.

### T1.b — add second `PortDefinition` and drop `.with_cfs_pml(...)`

Currently (lines ~3835-3842):

```rust
let port = PortDefinition {
    beta_mode: Box::new(fem_eig_006_beta_te10),
    modal_e_t: Box::new(fem_eig_006_modal_e_t_te10),
};

let solver = OpenBoundarySolver::new(&mesh, face_kinds, vec![port], MaterialDatabase::new())?
    .with_coupled_whitney(true)
    .with_cfs_pml(pml_config, pml_classes);
```

Replace with:

```rust
let port_0 = PortDefinition {
    beta_mode: Box::new(fem_eig_006_beta_te10),
    modal_e_t: Box::new(fem_eig_006_modal_e_t_te10),
};
// Phase 4.fem.eig.3.5.3 W1: +x port shares TE_{10} modal basis with
// the -x driving port (geometric translation along x preserves the
// modal shape).
let port_1 = PortDefinition {
    beta_mode: Box::new(fem_eig_006_beta_te10),
    modal_e_t: Box::new(fem_eig_006_modal_e_t_te10),
};

let solver = OpenBoundarySolver::new(
    &mesh,
    face_kinds,
    vec![port_0, port_1],
    MaterialDatabase::new(),
)?
.with_coupled_whitney(true);
```

The `.with_cfs_pml(pml_config, pml_classes)` builder chain drops
off entirely. The `pml_config: yee_fem::PmlConfig` function
parameter becomes unused (kept for source-compatibility with the
v3.5.2 ablation binary; tag with `let _ = pml_config;` or
`#[allow(unused_variables)]` and a doc-comment note).

## Risks + escape hatches

- **T2 smoke test produces NaN/Inf on wave-port modal projection.**
  Spec §7 (c) mesh-anisotropy mitigation: refine cavity to
  `(16, 5, 3)` and re-measure. If still NaN, escape-hatch and
  surface as a Phase 4.fem.eig.2 E2 wave-port robustness finding.

- **T3 measures `|S_{11}| > 0.1` after W1.** Spec §7 (a) higher-
  order modal content mitigation: add explicit TE_{20} / TE_{01}
  modes to the +x `PortDefinition`. If the multi-mode extension
  retires the gate, ship the v3.5.3 with multi-mode ports;
  otherwise queue Phase 4.fem.eig.3.5.4 for the rotated-PML W2
  hypothesis.

- **T3 regresses fem-eig-003.** Should not happen — W1 is
  fem-eig-006-driver-specific and fem-eig-003 uses a separate
  driver. If observed, the SSSSSSSSS S3 PmlConfig defaults were
  silently broken; escape-hatch and surface.

- **Blocked > 15 min on any step.** Surface and stop per
  CLAUDE.md §5; do **not** weaken any tolerance, do **not** patch
  outside the declared lane.
