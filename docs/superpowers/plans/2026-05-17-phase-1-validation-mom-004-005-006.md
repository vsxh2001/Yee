# Phase 1 Validation — mom-004 / mom-005 / mom-006 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to drive this plan task-by-task.

**Goal:** Land three published-benchmark microstrip validation cases on the existing MoM solver, in increasing order of complexity: Wilkinson divider (`mom-004`), branch-line hybrid (`mom-005`), and Swanson 5-pole hairpin BPF (`mom-006`). All three share substrate-stack synthesis machinery and the multi-port S-matrix path; they differ in port count (3 / 4 / 2), lumped elements (1 / 0 / 0), and mesh size (~1k / ~1k / ~5–10k RWG DoFs).

**Companion specs:**

- `docs/superpowers/specs/2026-05-17-phase-1-validation-mom-004-wilkinson-design.md`
- `docs/superpowers/specs/2026-05-17-phase-1-validation-mom-005-branch-line-hybrid-design.md`
- `docs/superpowers/specs/2026-05-17-phase-1-validation-mom-006-swanson-hairpin-bpf-design.md`

**Architecture:** A shared `microstrip_synthesis` module in `crates/yee-mom/src/` provides Hammerstad-Jensen width / εr_eff synthesis. Each validation case adds a geometry struct in `crates/yee-validation/src/fixtures/` and a `run_mom_00X` entry point in `crates/yee-validation/src/lib.rs`. mom-004 adds a `LumpedZSpec` plumbed via Schur reduction in `crates/yee-mom/src/solve.rs`; mom-005 and mom-006 reuse the multi-port machinery without adding to it.

## Step 1 — Shared microstrip synthesis module

**Files:** `crates/yee-mom/src/microstrip_synthesis.rs`, `crates/yee-mom/src/lib.rs`.

- [ ] Implement Hammerstad-Jensen formulae:
  - `eps_r_eff(w_over_h: f64, eps_r: f64) -> f64` (Eq. 4.1.5 in Hammerstad-Jensen 1980).
  - `z0_from_w(w_over_h: f64, eps_r: f64) -> f64` (Eq. 4.1.3–4.1.4).
  - `w_from_z0(z0: f64, eps_r: f64) -> f64` — inverse synthesis via Newton's method on `z0_from_w`; converges in ≤ 8 iters from `w_over_h = 1` initial guess.
  - `guided_wavelength(f_hz: f64, w_over_h: f64, eps_r: f64) -> f64` = `c / (f * sqrt(eps_r_eff))`.
- [ ] Unit tests against Pozar §3.8 Table 3.6 spot values:
  - FR-4 (εr=4.4): `Z₀(w/h=1.96) ≈ 50.0` (±0.5%).
  - FR-4: `Z₀(w/h=1.06) ≈ 70.7` (±0.5%).
  - RT/Duroid 6006 (εr=6.15): `Z₀(w/h=1.50)` matches published; verify against the substrate datasheet.
- [ ] Doc all public fns with `#![warn(missing_docs)]` already enforced workspace-wide.

Estimated LOC: ~180 (60 implementation + 120 tests). Verification: `cargo test -p yee-mom --lib microstrip_synthesis::tests` exits 0 in `< 1 s`.

## Step 2 — `LumpedZSpec` + Schur reduction (mom-004 prerequisite)

**Files:** `crates/yee-mom/src/lib.rs`, `crates/yee-mom/src/solve.rs`.

- [ ] Add `pub struct LumpedZSpec { pub node_a: u32, pub node_b: u32, pub z_ohms: Complex64 }` to `crates/yee-mom/src/lib.rs`. `node_a` / `node_b` reference RWG basis indices (not mesh vertex ids — basis indices are what the impedance matrix is indexed on).
- [ ] Extend `PlanarMoM::run` signature with `lumped: &[LumpedZSpec]`. The empty-slice path is **bit-for-bit identical** to the pre-mom-004 behavior; this is a regression-test gate.
- [ ] In `solve.rs`, after `impedance_matrix` fill, if `lumped.is_empty()` proceed as before. Otherwise, for each `LumpedZSpec`:
  - Add `+1/z_ohms` and `−1/z_ohms` admittance terms at the four `(node_a, node_b)` cross-positions of the Z matrix.
  - This is the standard "stamp" for a two-terminal lumped element on a basis-function nodal description; no Schur block reduction needed for a simple 2-terminal Z (the spec mentions "Schur" for generality with k-terminal elements, but a 2-terminal lumped Z is just an admittance stamp).
- [ ] Unit tests:
  - `lumped_empty_slice_bit_for_bit_unchanged` — `dipole_z_at_resonance` with `lumped: &[]` matches the prior reference output to `1e-12`.
  - `lumped_resistor_stamp_short_circuit` — a `LumpedZSpec { z_ohms: 1e-9 }` short between two ports of a 2-port test fixture forces `S₂₁ ≈ 0 dB` (path shorted).
  - `lumped_resistor_stamp_open_circuit` — `z_ohms = 1e9` recovers the unloaded S-matrix to `1e-6` relative.

Estimated LOC: ~120 (40 implementation + 80 tests). Verification: `cargo test -p yee-mom --release solve::tests::lumped` exits 0 in `< 30 s`. **mom-001 must remain green** (`cargo test -p yee-mom --release dipole_z_at_resonance`).
