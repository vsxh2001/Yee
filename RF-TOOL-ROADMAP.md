# RF Tool Roadmap (R.*) — from design demonstrations to a full RF design tool

**Date opened:** 2026-07-07
**Companion to:** `ENGINE-STUDIO-ROADMAP.md` (the engine + design loops this builds on),
`FILTER-DESIGN-ROADMAP.md`, `ROADMAP.md`.

The engine now *designs* — filters (S.8–S.12: synthesized → verified → refined to 1.0 %
cutoff error) and antennas (A.0–A.3: matched to −25.7 dB by measurement). This track
closes the gaps between "demonstrates design" and "a tool people design real hardware
with", in the order assessed on 2026-07-07: **(1) losses + vias, (2) complex S-matrix +
Touchstone export, (3) GPU parity for the design flows, (4) BPF end-to-end with
surrogate BO, (5) the studio spec→loop→export flow.**

Conventions as everywhere in this repo: every phase ships behind a machine-checkable
validation gate against a strong reference; walking-skeleton first; ADRs for decisions.

| Phase | Scope | Gate | Status |
|-------|-------|------|--------|
| **R.0** | **Dielectric loss on the board path**: substrate `tan δ` → per-cell σ (`σ = 2π f_ref ε₀ ε_r tan δ`, the standard single-frequency loss map — FDTD σ is frequency-flat, documented) riding the existing S.5 `sigma_cells` protocol field; `yee_voxel::substrate_sigma_cells` helper | `engine-loss-001`: measured dielectric attenuation α (forward-wave ratio between two directional probe triples ~3 λ_g apart) vs the **Pozar closed form** — **α = 4.328 vs 4.323 Np/m, 0.1 %** (±5 % gate) | **SHIPPED** (ADR-0194) |
| **R.0b** | Conductor loss: skin depth (~1–2 µm) is unresolvable at dx = 0.3 mm, so a volumetric σ_Cu is wrong by construction — needs a **surface-impedance boundary** on masked cells (own multi-step phase; do NOT fake it with bulk σ) | Ohmic attenuation of a line vs the closed-form α_c; the yee-fdtd skin-depth machinery is the reference | queued (after R.2) |
| **R.1** | **Vias**: vertical PEC columns ground→trace (`yee_voxel::with_via`, extends/attaches `pec_mask_ez`; protocol already carries the mask) | `engine-via-001` (differential, 3 runs): control notch **−45.1 dB @ 4.875 GHz**; with the via the same frequency reads **−1.3 dB** — the notch annihilated per TL theory | **SHIPPED** (ADR-0194) |
| **R.2** | **Complex 2-port S-matrix + Touchstone export**: `sparams::forward_transfer` (complex plane-to-plane `T = fwd_B/fwd_A`, backward wave AND the port-to-port multi-bounce resonance factor cancel in the ratio) + `sparams::complex_reflection` (complex Γ = bwd/fwd — the DUT's reflection for a **one-port** DUT, per the A.1/A.3 usage); measured through-line exported `.s2p` via `yee_io::touchstone` (S11 = S22 = 0, exact for a uniform line in its own reference impedance) with **per-frequency passivity enforcement** (σ_max closed-form `max\|a±b\|` for the symmetric fill; correction asserted ≤ 0.5 dB) | `engine-sparams-002`: unwrapped-phase slope vs HJ closed form `−2π d √ε_eff/c` — **2.91 %** (±5 % gate); worst \|T\| **+0.24 dB** (±1 dB); `.s2p` write→read round-trip at writer precision | **SHIPPED** (ADR-0195) |
| **R.2b** | **Measured through-line S11**: de-embedding/calibration so the exported S11 is the DUT's, not the fixture's (the ADR-0195 negative result: on a THRU, plane-A Γ is the load-port's reflection, and window-truncated multi-bounce pushes raw \|Γ\| to 2.4 — needs ring-down-complete windows or shorter lines + port calibration) | complex Γ of a known one-port (via-shorted line: Γ = −1 rotated by 2βl) vs TL theory, magnitude AND phase | queued |
| **R.3** | **GPU parity for the design flows**: `apply_aperture_ports` WGSL kernel (one invocation per port, verbatim CPU algorithm, append-only drive-buffer extension) + per-face CPML (`faces_mask`, 6 bits, bit `2·axis+side`); both `Unsupported` rejections removed — `backend: "gpu"` runs the filter/antenna/board flows, `auto` stops falling back. Protocol NTFF stays CPU-only (own increment; yee-compute's on-GPU DFT is E.5b) | `compute-015` (aperture board, 400 steps, probe series vs CPU): **L2 ≤ 1.2e−7, L∞ ≤ 1.6e−7** (gates 1e−3/5e−3); `compute-016` (open-top per-face CPML, full fields): **E ≤ 2.1e−7, H ≤ 1.7e−4** (gates 1e−4/1e−3), absorption ‖H‖ 2.5e−5 vs 1.0e−2 PEC — both on llvmpipe + the GPU nightly | **SHIPPED** (ADR-0196) |
| **R.4** | **BPF end-to-end + multi-knob BO** (= the deferred F1.2.1 core). **R.4a SHIPPED**: per-section hairpin geometry (`hairpin_bpf_sections`), qe→tap closed form (`tap_offset_from_qe`, `Qe = (π/2)(Z0/Zr)/cos²(πt/L)`, gate `tap-qe-001`), **fold-corrected arm length** `arm = (λ_g/2 − fold)/2` (`dimension_hairpin_with_fold`; the λ_g/4 form left resonators long by the bend path), degenerate-fold + tap-realizability guards. **Measured negative result** (3 instrumented runs + probe forensics): the closed-form seed's \|Γ_in\| ≈ 1 across the band — resonance ~+17 % high, tap coupling ≪ designed qe, S21 peak −19 dB — no closed form on this stack repairs it, so the full-wave gate IS the BO gate | `engine-bpf-bo-001`: 1 reference + 12 DUT solves; `yee_surrogate::bo::minimize` over (arm, tap, gap) scales minimizing RMS misfit to `coupling_matrix_s_params`. First converged run: misfit **28.48 → 24.59 dB RMS (−3.9 dB)**, best evals at the gap-scale grid floor — the loop works and the scenario is coupling-limited. Gate asserts the machinery: improvement ≥ 2 dB RMS on a live ≥ 15 dB-dynamic-range measurement | **SHIPPED** (machinery + feasibility finding; ADR-0197) |
| **R.4c** | **Fine-grid BPF close-out**: the same `engine-bpf-bo-001` harness at dx ≈ 0.1 mm (~8× cells — GPU-nightly scale, which R.3 enables), where sub-0.4 mm gaps resolve and the designed k/qe become realizable | "BO closes centre frequency + bandwidth to spec" becomes assertable: passband peak ≥ −6 dB, centre ±5 %, BW ±30 % | queued (GPU nightly) |
| **R.5** | **Studio spec→design→export** (walking skeleton SHIPPED): `design_filter` Tauri command (`design_filter_impl` core) — spec → synthesize → R.4a dims (fold-corrected, qe→tap, per-section) → layout → `coupling_matrix_s_params` response → `.s2p` + copper/outline Gerber artifacts, instant; React `FilterDesignPanel` (spec form, dims readout, Blob-download exports) + `SparamPlot` (first response-vs-frequency view). **R.5b SHIPPED** (ADR-0199): `yee_engine::board` promotes the gates' copy-pasted measurement fixture to a shared library (`two_port_board_job` — the studio measures exactly the way the gates do); `verify_filter` command runs reference + DUT solves with `verify://progress` phase streaming and returns measured vs designed \|S21\| (React overlay plot). Antenna specs + a studio BO button remain follow-ons (the latter waits on R.4c) | `studio-design-e2e-001` (headless command-layer e2e, in the `studio-build` CI job): scripted flow; `.s2p` **byte-stable** through `yee_io::touchstone` read→re-render; Gerbers **byte-identical** to `yee_export`'s output; band-pass response asserts; unrealizable-spec error path. Plus vitest DOM gates (`sparam.test.tsx`). **R.5b**: `studio-verify-e2e-001` (reduced-fidelity pipe e2e): phase-ordered progress; the **exact-identity case** — verifying a line against itself must read \|S21\| = 0 dB to < 1e−9 (two bit-identical solves); builder-error surfacing | **SHIPPED** (walking skeleton; ADR-0198) |
